# 🛠️ DOE Tool 메뉴 설계서 (Version 1)

## 1. 설계 목표
- 업무용 데이터 분석 + 실험계획법(Design of Experiments) 핵심 기능을 **빠르게 제공**하면서도, 장기적으로 확장할 수 있는 메뉴 구조를 정의한다.
- 초기(Ver 1.0)에는 실제 구현이 완료된 메뉴만 **활성화**(Enable)하여 사용자 혼란을 최소화한다.
- 향후 기능 개발 시 **메뉴만 선반영 → 단계별 활성화** 방식(feature toggle)으로 배포·테스트 부담을 줄인다.

> **표기 규칙**  
> 🟢 활성 (Ver 1.0)  |  🟡 예정 (Ver 1.x~2.x)  |  🔴 장기 (Ver 3.x 이후)  
> 괄호 안은 **예정 활성화 버전**

## 4. UI/UX 표준
1. **비활성 메뉴**는 회색 처리(Grey-out)하여 존재를 알리되 클릭 불가
2. 기능 구현 완료 후 QA 통과 → 체크리스트 업데이트 → 활성화
3. 단축키는 가능한 표준(Windows 기준) 우선 할당
