# 메뉴 구조 재설계 및 용도 정의

이 문서는 통계 분석 도구의 전체 메뉴 구조를 정의하고, 각 메뉴의 핵심 용도와 필요성을 설명합니다.

---

## 1. 메뉴별 주요 기능 및 용도

애플리케이션의 메뉴는 사용자의 작업 흐름에 따라 직관적으로 구성되었습니다. 각 메뉴는 명확한 목적을 가지며, 사용자가 특정 작업을 수행해야 할 때 어떤 메뉴를 선택해야 하는지 안내합니다.

### 📂 **파일 (File) 메뉴**

-   **핵심 용도**: **프로젝트 및 데이터 생명주기 관리**
-   **언제 사용하는가?**:
    -   새로운 분석 프로젝트를 시작하거나(`새 프로젝트`), 기존 작업을 불러올 때(`프로젝트 열기`).
    -   진행 중인 작업을 저장하거나(`저장`, `다른 이름으로 저장`), 외부로 내보낼 때(`데이터 내보내기`).
    -   분석할 데이터를 Excel, CSV 등의 외부 파일에서 가져올 때(`데이터 가져오기`).
    -   애플리케이션을 종료할 때.
-   **설명**: 파일 메뉴는 분석 작업의 시작과 끝을 책임집니다. 데이터나 프로젝트 파일을 관리하는 모든 기능이 여기에 위치합니다.

### 🔬 **DOE (Design of Experiments) 메뉴**

-   **핵심 용도**: **실험 계획 데이터 생성**
-   **언제 사용하는가?**:
    -   어떤 요인이 결과에 영향을 미치는지 효율적으로 찾기 위한 실험 계획을 세울 때.
    -   `완전요인설계`, `부분요인설계` 등을 통해 분석에 필요한 최소한의 실험 조합을 만들고 싶을 때.
    -   제품이나 공정의 최적 조건을 찾기 위해 `반응표면법(RSM)`과 같은 고급 실험 계획이 필요할 때.
-   **설명**: DOE 메뉴는 **데이터를 분석하는 곳이 아니라, 분석할 데이터를 체계적으로 설계하고 생성하는 곳**입니다. 사용자는 이 메뉴를 통해 통계적으로 유의미한 실험 데이터 테이블을 만들 수 있습니다.

### 📊 **분석 (Analysis) 메뉴**

-   **핵심 용도**: **데이터의 통계적 관계 및 의미 해석**
-   **언제 사용하는가?**:
    -   준비된 데이터(DOE로 생성했거나, 직접 가져온 데이터)의 통계적 특성을 파악하고 싶을 때 (`기초 통계`).
    -   그룹 간의 평균 차이가 유의미한지 검증하고 싶을 때 (`분산분석`).
    -   변수들 간의 인과관계나 예측 모델을 만들고 싶을 때 (`회귀분석`).
    -   데이터를 특정 기준에 따라 분류하거나 요약하고 싶을 때 (`다변량 분석`).
-   **설명**: 분석 메뉴는 데이터에 숨겨진 패턴, 관계, 차이 등을 통계 모델을 통해 증명하고 해석하는 모든 기능의 집합입니다. 각 하위 메뉴는 특정 통계 분석 기법을 수행합니다.

### 📈 **차트 (Chart) 메뉴**

-   **핵심 용도**: **데이터의 탐색적 시각화 및 맞춤형 시각화**
-   **언제 사용하는가?**:
    -   본격적인 통계 분석에 앞서 데이터의 분포나 변수 간 관계를 시각적으로 탐색하고 싶을 때. (예: `산점도`, `히스토그램`)
    -   특정 통계 분석에 종속되지 않는 일반적인 목적으로 데이터를 시각화하고 싶을 때.
    -   분석 결과로 자동 생성된 차트 외에, 사용자가 직접 변수를 선택하여 맞춤형 차트를 만들고 싶을 때.
-   **설명**: 차트 메뉴는 사용자가 자유롭게 데이터를 탐색하고 이해하는 데 도움을 주는 도구입니다. 아래 "차트 메뉴의 필요성" 섹션에서 더 자세히 설명합니다.

### 💾 **데이터 (Data) 메뉴**

-   **핵심 용도**: **분석 전 데이터 준비 및 가공 (Data Preprocessing)**
-   **언제 사용하는가?**:
    -   데이터를 정렬하거나, 특정 조건에 맞는 데이터만 필터링하고 싶을 때.
    -   분석에 용이하도록 데이터의 형태를 바꾸거나(`로그 변환`, `표준화`), 파생 변수를 만들 때.
    -   결측값을 처리하거나, 데이터의 타입을 변경하는 등 데이터를 '정제'하는 작업이 필요할 때.
-   **설명**: 데이터 메뉴는 원본 데이터를 분석에 적합한 형태로 '요리'하는 과정입니다. 아무리 좋은 분석 도구도 데이터가 제대로 준비되지 않으면 의미가 없으므로, 데이터 메뉴는 분석의 정확도를 높이는 데 필수적입니다.

### 🛠️ **도구 (Tools) 메뉴**

-   **핵심 용도**: **애플리케이션 환경 설정 및 보조 기능**
-   **언제 사용하는가?**:
    -   애플리케이션의 테마, 폰트, 기본 저장 경로 등 환경을 설정하고 싶을 때 (`환경설정`).
    -   열려있는 모든 차트나 결과창을 한 번에 정리하는 등 작업 편의를 위한 부가 기능이 필요할 때.
-   **설명**: 도구 메뉴는 특정 데이터나 분석에 종속되지 않고, 애플리케이션 전반에 영향을 미치는 설정 및 유틸리티 기능을 제공합니다.

### 🆘 **도움말 (Help) 메뉴**

-   **핵심 용도**: **사용자 지원 및 정보 제공**
-   **언제 사용하는가?**:
    -   애플리케이션의 버전, 라이선스 정보가 궁금할 때 (`정보`).
    -   각 기능의 사용법이나 통계 용어에 대한 가이드가 필요할 때.
-   **설명**: 사용자가 애플리케이션을 사용하면서 궁금한 점이 생겼을 때 참고할 수 있는 모든 정보를 담고 있습니다.

## 2. 차트 메뉴의 분리 필요성

"각 `분석` 메뉴에서 추천 시각화가 제공되는데, 왜 `차트` 메뉴가 별도로 필요한가?" 라는 질문은 매우 중요하며, 그 이유는 다음과 같습니다.

### **역할의 분리: 분석 결과 vs. 탐색 도구**

1.  **분석 메뉴의 차트 = 분석의 '결과'**
    -   `분석` 기능 내에서 생성되는 차트(예: 회귀분석의 잔차 그림, 분산분석의 상호작용 그림)는 해당 **통계 모델의 가정을 검증하거나 결과를 해석하기 위한 목적**을 가집니다.
    -   이 차트들은 분석 모델에 강하게 종속되어 있으며, 분석 없이는 의미가 없거나 만들어낼 수 없습니다. 즉, 이것은 **분석 리포트의 일부**입니다.

2.  **차트 메뉴의 차트 = 자유로운 '탐색' 도구**
    -   `차트` 메뉴는 **탐색적 데이터 분석(Exploratory Data Analysis, EDA)**을 위한 도구입니다.
    -   사용자는 특정 통계 모델을 염두에 두지 않고도, 두 변수를 `산점도`에 그려보거나, 각 변수의 `히스토그램`을 그려보며 데이터에 대한 직관과 통찰을 얻을 수 있습니다.
    -   이러한 탐색 과정은 어떤 분석을 수행할지 결정하는 데 중요한 역할을 합니다.

### **사용자 워크플로우 지원**

일반적인 데이터 분석가는 다음과 같은 순서로 작업을 진행하며, `차트` 메뉴는 이 흐름에 필수적입니다.

-   **1단계 (데이터 로드)**: `파일` 메뉴로 데이터를 불러온다.
-   **2단계 (데이터 탐색)**: `차트` 메뉴를 사용해 데이터의 분포(히스토그램, 상자그림)나 변수 간 관계(산점도)를 시각적으로 확인하며 데이터의 특징을 파악한다.
-   **3단계 (데이터 전처리)**: `데이터` 메뉴를 사용해 탐색 단계에서 발견한 이상치나 결측값을 처리하고, 필요하면 변수를 변환한다.
-   **4.단계 (모델링 및 분석)**: `분석` 메뉴에서 탐색 결과에 기반한 적절한 통계 모델을 선택하여 분석을 수행한다. 분석 결과로 제공되는 **전용 차트**를 통해 모델을 진단하고 결과를 해석한다.
-   **5단계 (결과 보고)**: `차트` 메뉴를 다시 사용하여 분석 결과를 보완하는 **맞춤형 시각화 자료**를 생성한다.

### **결론**

`분석` 메뉴의 시각화와 `차트` 메뉴의 시각화는 **목적**과 **사용 시점**이 다릅니다.
-   **분석 메뉴의 차트**: **"이 분석 결과는 타당한가?"** 에 답하기 위한 진단용/해석용 차트.
-   **차트 메뉴의 차트**: **"이 데이터는 어떤 특징을 가지고 있는가?"** 에 답하기 위한 탐색용/범용 차트.

따라서 `차트` 메뉴를 독립적으로 제공하는 것은 사용자가 데이터를 더 깊이 있고, 자유롭게 이해하도록 돕는 필수적인 설계입니다.
